입력 양식 만들기

	클라이언트 정보를 서버에서 가져갈 수 있도록 환경 설정
	[form태그]
	 0. form태그
	   1) action속성 : 입력된 데이터가 submit될 시 데이터를 보낼 파일 혹은 url 명시
	   2) method속성 : 전송 방식 설정
	      - get : default값. 전송시 입력값이 도메인에 그대로 보임(보안 취약)
	      - post : 파일 헤더에 정보값이 암호화 되어 저장, 도메인은 변화 없음(로그인 등 보안 필요시에 사용)
	   
	 1.input 태그 : 실질적으로 데이터를 입력받을 양식을 만들기 위한 테그
	   1) type속성 : 입력받을 데이터 타입 설정
	      - text : 텍스트
	      - password : 비밀번호
	      - file : 파일입력을 위한 파일 선택 버튼 생성 후 옆에 파일 경로 표시
	               (파일 없을시 경로에 '선택된 파일 없음' 표시)
	      - checkbox : 체크박스
	         * value 속성에 입력하는 값 : DB에 입력될 값. 즉 사용자가 보는 값이 아님.
	           ㄴ 사용자가 보는 값은, input테그 뒤에 작성
	      - radio : 라디오 버튼
	      	 * value 속성에 입력하는 값 : DB에 입력될 값. 즉 사용자가 보는 값이 아님.
	           ㄴ 사용자가 보는 값은, input테그 뒤에 작성
	      - 전송 관련 type속성 지정 값들
	    	  ① button : 서버에 있는 jsp를 통해 등록에 관한 프로그래밍을 추가해주어야 함.
	        	 ㄴ 기능을 구현하지 않을 시 아무 작동도 하지 않음.
	    	  ② submit : 입력된 데이터들을 지정된 파일 혹은 url로 submit함
	    	    cf) button 태그 : input 태그의 submit 속성과 같은 기능
	    	  ③ image : 글자가 써진 버튼 대신 이미지를 띄움.
	   2) name 속성 : 입력된 데이터 값의 변수명 설정(DB에서 사용할 변수명)
	       - name 속성 지정된 데이터를 입력하지 않는 경우, 다음으로 넘어가지지 않음
	   3) value 속성: 기본값 지정
	   4) placeholder 속성 : 데이터 입력 전 안내글 표시
	   5) maxlength 속성 :최대 입력 글자수 지정
	   6) size 속성 : 블럭 크기 설정
	   7) readonly 속성 : 읽기 전용으로 변환
	   8) disabled 속성 : 비활성화(서버에서 request도 불가능)
	   9) required 속성 : 마우스 오버시, 혹은 미입력후 저장 시도시 입력 안내글 표시(default : 이 입력란을 입력하세요)
	   10) checked 속성 : 체크박스와 라디오버튼에서 default로 체크해서 보여줄 값에 설정

	 2.textarea 태그 : 문자들을 입력받기 위한 테그
	   1) memo 속성 : 여러줄의 문자를 입력받기
	   2) name 속성 : 입력된 데이터 값의 변수명 설정(DB에서 사용할 변수명)
	   2) cols 속성 : 가로 길이 설정
	   3) rows 속성  : 세로 길이(줄) 설정
	   4) (value) : 따로 속성을 쓰지 않고 시작테그와 끝테그 사이에 작성

	 3. button 태그 : input 태그의 submit속성과 기능이 같음. 시작태그와 끝태그 사이에 사용자에게 표시할 글자 작성.
